# Question Generation Quality Roadmap

> 用于跨会话跟踪“从素材生成试题”的质量优化进度，避免后续因上下文过长丢失目标和边界。

## Milestones

- [x] M0 固化执行框架：将里程碑清单落盘，作为后续实施基线。
- [x] M1 质量止损（第一轮）：
  - 恢复整套题核心校验
  - 暴露预览风险信号
  - 预览保存前端阻断
  - 后端保存接口拒绝重复题干和未通过校验的素材题
- [x] M2 素材输入质量（第一轮）：
  - 结构化切片替代固定字符硬切片
  - 禁止图片/视频/音频占位解析结果参与出题
  - 解析阶段不再用素材级 category 覆盖所有知识单元维度
  - 生成阶段补充知识单元标题/摘要/关键词上下文
- [x] M4 Prompt 约束（第一轮）：
  - 自定义用户提示词模板必须保留核心占位符
  - 降低生成温度，减少格式漂移
  - 收紧错误答案的自动修复逻辑
- [x] M3 知识片段选择与题量分配（第一轮）：
  - 由“前 N 个片段 + 平均分配”改为“按信息价值选取 + 按知识点价值分配”
  - 多素材按总题量统一规划，不再按素材数复制题量
  - 提示词预览与实际生成链路对齐到同一套选片/分配规则
- [x] M4 Prompt 机制（第二轮，第一轮落地）：
  - 已增加“知识点规划 -> 题目生成”的两阶段 prompt 结构，规划结果会注入最终出题提示词
  - prompt preview 已展示规划段，避免预览与实际生成逻辑脱节
  - 已增加更强的模型级结构化输出约束：优先使用 `response_format/json_schema`，不支持时自动降级
  - 已升级为“规划器 + 生成器”的双调用链路；规划器失败时会自动回退到本地确定性规划
- [x] M5 保存前质检与去重：
  - 已增加预览阶段自动 AI 质检，并在预览表中展示质检结论
  - 已增加同批预览题目的近重复/换皮检测，并在预览和保存时拦截
  - 已增加与历史题库的近重复比对，并在预览和保存时拦截
  - 已按题型细化规则质检，补强判断题/填空题/主观题的结构检查
- [x] M6 校准与指标：
  - 已增加预览阶段的难度/Bloom 后验校准，并输出逐题校准结果
  - 已增加难度偏差、Bloom偏差等质量指标统计
  - 已将结构化质量指标接入题库页概览卡片
  - 已将题库质量快照与趋势对比接入月报，支持跨月跟踪缺失 Bloom、缺失解析等问题
  - 已补充题库统计、预览校准、月报质量趋势等回归测试

## Next Recommended Step

当前路线图已完成。后续若继续推进，建议转向基于这些质量指标做自动化治理，例如：按月自动补标、低质量题批量复审、以及基于趋势的运营告警。

## Follow-up Changes

- 已新增存量素材“安全重解析”能力：
  - 提供 `/api/v1/materials/{material_id}/reparse` 入口，替换旧知识单元而不是继续叠加
  - 重解析前会解绑历史题目的 `source_knowledge_unit_id`，保留素材级来源
  - 对已向量化素材会清理旧向量，并按需要重建新向量
  - 对仍处于 OCR/ASR 占位文本的素材会拒绝重解析，避免误清洗线上数据
- 已新增素材建库的“知识片段策略”：
  - 默认仍为 `stable`，保持固定条件下的稳定选片
  - 新增 `coverage` 模式，会参考同一素材最近 3 次成功建库命中的知识单元，对高频片段做冷却降权
  - 新增 `material_generation_runs` / `material_generation_run_units` 历史表，记录每次成功预览实际命中的知识单元
  - 仅当素材预览通过质量门禁时才写入历史，避免失败预览污染后续冷却结果
  - prompt preview、AI 建议配比、题库预览三条链路已统一接入 `selection_mode`
