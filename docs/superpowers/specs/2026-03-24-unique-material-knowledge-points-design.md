# Unique Material Knowledge Points Design

**Date:** 2026-03-24

## Goal

让“从素材生成题库”在同一批次内只使用不同的知识点出题，并修复素材元信息门禁对固定单字 `章`、`节` 的误杀问题。

## Scope

本次只覆盖素材驱动的题库生成链路：

- 题库预览
- 正式建库
- Prompt Preview
- 建议题型配比与总题量

不修改自由生成，也不修改 `batch_generate_from_material(count_per_unit=...)` 这种按知识单元批量生成的旧链路。

## Decisions

### 1. 同批题库生成强制唯一知识点

- 同一批题库生成中，每个知识单元最多只允许承担 1 道题。
- 不再允许通过跨题型复用同一知识单元来凑题量。
- 如果请求题量大于当前素材可提供的去重后知识单元数量，系统直接失败，不进入 LLM 生成。

这意味着：

- 单选 5 题需要至少 5 个可用知识单元。
- 混合题型总量为 5 题时，也同样需要至少 5 个可用知识单元。
- “知识点重复”继续保留为生成后兜底校验，但不再依赖它发现主要问题。

### 2. 实际生成时自动扩展候选片段上限

- 对素材生成链路，实际选片上限会自动扩展到 `max(max_units, requested_total)`。
- 这样可以避免因为用户设置的 `max_units` 偏小而被无意义拦截。
- 如果扩展后仍不足，直接报明确错误。

### 3. Prompt Preview 与正式生成保持一致

- Prompt Preview 必须使用与正式生成一致的唯一知识点约束。
- 多素材 Prompt Preview 在按素材分配题量之后，需要对每个素材重新做一次“按请求题量扩容后的选片”，避免预览展示和真实生成不一致。

### 4. 建议题量受唯一知识点容量约束

- `suggest_question_distribution()` 生成建议题量时，建议总题量不能超过当前选中知识单元数量。
- 这保证系统推荐不会天然高于唯一知识点容量。

### 5. 修复素材元信息禁词误杀

- 从固定禁词中移除单字 `章`、`节`。
- 保留明确短语：
  - `根据本文`
  - `本书`
  - `该书`
  - `书中`
  - `出版社`
  - `ISBN`
  - `章节`
  - `第1章/第2章/...`
  - `第1节/第2节/...`
- 保留现有章节/作者/出版社等正则匹配。

这样可以避免把正常技术词如“节点”“环节”“调节”误判成素材章节引用。

## Error Handling

当唯一知识点容量不足时，统一返回：

`当前素材去重后仅有 X 个可用知识点，不足以生成 Y 道互不重复知识点的题目`

这个错误在 LLM 调用前抛出，避免无效消耗。

## Testing

需要覆盖三类测试：

1. 分配层：
   - 混合题型请求下，每个知识单元最多 1 题
   - 总题量与请求一致
2. 容量不足层：
   - 请求题量超过可用知识单元数时，预览直接失败
   - 不触发 LLM 调用
3. 门禁层：
   - `节点`、`环节`、`调节`、`多个节点环境中部署实例` 不再命中素材元信息
   - `第3章`、`作者：`、`出版社：` 仍然命中
