# Scores Table Actions Layout Design

## 背景

成绩管理页的个人列表和管理员列表都在“操作”列中直接放置多个链接按钮：

- 个人列表：诊断分析报告、复盘、手动评分（条件显示）、删除
- 管理员列表：诊断报告、下载证书、删除

当前 [Scores.vue](/home/ps/project/ai-literacy-platform/frontend/src/views/Scores.vue) 中两张表都没有为操作区提供抗挤压布局，列宽固定且表格未显式设置横向滚动，导致在较窄页面宽度下操作内容溢出单元格。

## 目标

在不改变操作文案、顺序和交互方式的前提下，让成绩管理页两张主列表的“操作”列在常见窗口宽度下不再溢出。

## 非目标

- 不把操作折叠进“更多”菜单
- 不改变按钮文案或权限逻辑
- 不调整诊断报告、复盘等详情区域
- 不重构其他表格（如英雄榜）

## 设计决策

### 1. 适度加宽“操作”列

两张主列表的“操作”列都上调宽度：

- 个人列表：为 4 个操作预留更稳定的水平空间
- 管理员列表：为 3 个操作预留稳定空间

仅靠增宽不能彻底解决极窄窗口问题，但这是最符合用户“适当调整表格宽度”诉求的主措施。

### 2. 为表格设置横向滚动

给个人列表和管理员列表增加 `scroll.x`，使表格在总列宽超过容器时横向滚动，而不是压缩到操作列溢出。

这能把问题限定在表格内部处理，不会把整页布局挤坏。

### 3. 操作区改为可换行的紧凑布局

保留所有操作直出，但让操作区容器支持自动换行。这样在较窄宽度下，按钮会折到下一行，而不是突破单元格边界。

实现上优先复用 Ant Design Vue 的 `a-space` 换行能力，只做轻量 CSS 收口，不另造复杂容器。

## 页面行为

- 宽屏下，操作仍尽量保持单行展示
- 中等宽度下，操作列变宽，表格允许横向滚动
- 更窄宽度下，操作按钮可以在列内换行
- 管理员列表和个人列表行为保持一致

## 验证方案

- 个人成绩列表“操作”列不再溢出
- 管理员成绩列表“操作”列不再溢出
- 按钮点击行为不受影响
- 前端构建通过，确保模板与样式修改没有破坏页面编译
